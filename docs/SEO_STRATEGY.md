## 4. Internal Linking Structure

**From home `/`**  
- To: `/features`, `/#pricing`, `/demo`, `/blog`.  
- Anchor: “Features”, “Pricing”, “Demo”, “Blog”.  
- Add 1–2 contextual links in hero or first section: “See distribution software features” → `/features`, “Get a demo login” → `/demo`.  
- Footer already links to features, pricing, demo, blog, location pages.

**From `/features`**  
- To: each `/features/[slug]` (card links), `/demo` (CTA strip).  
- Anchor: Feature title → slug; “Get demo” / “WhatsApp” → `/demo`.

**From each `/features/[slug]`**  
- To: `/features` (back), `/demo`, WhatsApp.  
- Optional: one link to related feature or location (e.g. credit-control → `/credit-management-software-uae`).

**From location pages**  
- To: `/features` (e.g. “See HexaBill features”), `/demo` (CTA).  
- Optional: 1–2 links to related location (e.g. Dubai → Abu Dhabi, Sharjah) or topic (e.g. Dubai → `/vat-invoice-software-uae`).

**From `/blog` and `/blog/[slug]`**  
- To: `/features` (e.g. “See how HexaBill handles this”), `/demo`, relevant `/features/[slug]` when post has `featureSlug`.  
- Anchor: “HexaBill features”, “Get demo login”, feature name.

**From `/pricing` and `/demo`**  
- To: `/features` (e.g. “See features”), `/` (home).  
- CTA: “Get Demo Login” / “WhatsApp” (same as site-wide).

**When you add `/compare`**  
- Link from: `/` (trust or “Why HexaBill” block), `/features`, `/pricing`.  
- Link to: `/features`, `/demo`.

## 5. Six-Month Content Authority Plan

Realistic: rankings and organic volume build over 3–6+ months. Focus on on-page quality, internal links, and a few new assets.

**Month 1**  
- On-page: Ensure every existing page has primary keyword in title/H1 and 2–3 supporting terms in body.  
- Internal links: Add 2–3 contextual links from home to features/demo; from location pages to features and demo.  
- No new pages yet; fix meta descriptions and titles where thin.

**Month 2**  
- Publish 1–2 blog posts (e.g. VAT mistakes Gulf, credit control for distributors). Link to `/features` and `/demo` and relevant feature slug.  
- Optional: Add comparison page outline to roadmap and draft copy (see Section 6 below).

**Month 3**  
- Launch comparison page `/compare` or `/vs-zoho-tally-quickbooks` (see Comparison Page Outline below). Link from home, features, pricing.  
- Publish 1 blog post. Internal link from blog to features and demo.

**Month 4**  
- First micro-niche page (e.g. Frozen Food Billing Software UAE or Route Sales Software GCC) using playbook Section 7 outlines. Link from `/features`, `/billing-software-dubai`, and home if you add a “By industry” or “Resources” block.  
- 1 blog post.

**Month 5**  
- Second micro-niche page.  
- 1 blog post.  
- Review all location pages: add 1–2 supporting keywords and one internal link to features/demo if missing.

**Month 6**  
- Third micro-niche page.  
- 1 blog post.  
- Audit: primary keyword and internal links on every page. Fix thin content or duplicate targets.

**Ongoing**  
- Every new blog or page: link to `/features` and `/demo`. Use anchor text that includes a supporting keyword where natural.  
- No expectation to “rank #1” in 6 months; goal is topical coverage and a clear path from any page to demo.
